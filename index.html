10 · Error Handling
How are supplier outages detected?
5xx HTTP status code is returned when sending requests to them.
Is retry logic implemented at the connector layer or expected upstream (HX)?
Done — Retry mechanism is implemented on IG for async booking flow.
Are there suppliers where availability/booking failures are common or require special handling?
Mostly parking suppliers like QuickParking (require VRN when entry date is close to current datetime) and ilogs (VRN format; same VRN cannot be used to book multiple parking lots in different entry/exit datetimes).
11 · Data Completeness & Constraints
Are all supplier price/availability changes captured and pushed upstream?
Yes — Price mismatch event, availability event, and no-availability event are published.
Are there suppliers where we rely on polling vs event-driven updates?
Yes — TransferZ for now. IG is moving from long polling to webhooks to track TransferZ's journey changes.
Are there gaps in delta detection for pricing updates?
Yes — Price mismatch event will be published.
Are there suppliers that would require significant rework to align with the target gateway model?
No
